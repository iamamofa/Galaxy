Package Management with Conda

Managing Bioinformatics Software

Conda is a powerful package manager that simplifies installing bioinformatics software and managing dependencies. It eliminates the "dependency hell" problem by creating isolated environments.

What is Conda?

  • Cross-platform package manager (works on Linux, macOS, Windows)
  • Creates isolated environments to avoid conflicts
  • Manages both Python and non-Python packages
  • Bioconda channel provides 8000+ bioinformatics packages

Installing Conda

Option 1: Miniconda (Recommended)

Lightweight installer with just conda and Python

# Download Miniconda installer
$ wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh

# Run installer
$ bash Miniconda3-latest-Linux-x86_64.sh

# Initialize conda
$ conda init bash
$ source ~/.bashrc

Configuring Conda Channels

Add bioinformatics-specific channels:

$ conda config --add channels defaults
$ conda config --add channels bioconda
$ conda config --add channels conda-forge
$ conda config --set channel_priority strict

Creating and Managing Environments

Create a New Environment

# Create environment for cholera analysis
$ conda create -n cholera_analysis python=3.9

# Activate the environment
$ conda activate cholera_analysis
(cholera_analysis) $

Installing Bioinformatics Tools

Install Common Tools

# Quality control tool
$ conda install -c bioconda fastqc

# Read trimming
$ conda install -c bioconda trimmomatic

# Genome assembler
$ conda install -c bioconda spades

# Genome annotation
$ conda install -c bioconda prokka

Practical Exercise

Exercise: Set Up Analysis Environment

$ conda create -n cholera python=3.9
