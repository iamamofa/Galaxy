Nextflow for Bioinformatics

Build Scalable and Reproducible Pipelines

Nextflow is a powerful workflow management system that enables scalable and reproducible bioinformatics pipelines. It supports Docker, Singularity, and major cloud platforms.

Why Nextflow?

  • Portable - runs on local computers, clusters, and cloud
  • Reproducible - supports containers and conda environments
  • Scalable - parallel execution across multiple nodes
  • Resumable - continue from where you left off if something fails

Installing Nextflow

Quick Installation

# Install Nextflow
$ curl -s https://get.nextflow.io | bash

# Make it executable
$ chmod +x nextflow

# Move to PATH
$ sudo mv nextflow /usr/local/bin/

# Test installation
$ nextflow info

Nextflow Basics - DSL2 Syntax

Simple Nextflow Script

Create a file called hello.nf:

#!/usr/bin/env nextflow
nextflow.enable.dsl=2

// Simple process
process sayHello {
output:
stdout

script:
"""
echo 'Hello Bioinformatics!'
"""
}

// Workflow
workflow {
sayHello()
}
# Run the workflow
$ nextflow run hello.nf

Cholera Analysis Pipeline

Complete Analysis Workflow

Create cholera_pipeline.nf:

#!/usr/bin/env nextflow
nextflow.enable.dsl=2

// Parameters
params.reads = "data/*_{1,2}.fastq.gz"
params.outdir = "results"

// Quality Control Process
process FASTQC {
container 'biocontainers/fastqc:v0.11.9_cv8'
publishDir "${params.outdir}/qc", mode: 'copy'

input:
tuple val(sample_id), path(reads)

output:
path "*.html"
path "*.zip"

script:
"""
fastqc ${reads}
"""
}

// Trimming Process
process TRIMMOMATIC {
container 'biocontainers/trimmomatic:0.39'

input:
tuple val(sample_id), path(reads)

output:
tuple val(sample_id), path("*_paired_*.fastq.gz")

script:
"""
trimmomatic PE ${reads[0]} ${reads[1]} \\
${sample_id}_paired_1.fastq.gz ${sample_id}_unpaired_1.fastq.gz \\
${sample_id}_paired_2.fastq.gz ${sample_id}_unpaired_2.fastq.gz \\
LEADING:3 TRAILING:3 SLIDINGWINDOW:4:15 MINLEN:36
"""
}

// Assembly Process
process SPADES {
container 'quay.io/biocontainers/spades:3.15.5'
publishDir "${params.outdir}/assembly", mode: 'copy'

input:
tuple val(sample_id), path(reads)

output:
tuple val(sample_id), path("${sample_id}/contigs.fasta")

script:
"""
spades.py -1 ${reads[0]} -2 ${reads[1]} -o ${sample_id} --careful
"""
}

// Main Workflow
workflow {
reads_ch = Channel.fromFilePairs(params.reads)
FASTQC(reads_ch)
trimmed = TRIMMOMATIC(reads_ch)
SPADES(trimmed)
}

Run the Pipeline

# Run with Docker
$ nextflow run cholera_pipeline.nf -with-docker

# Resume from where it stopped
$ nextflow run cholera_pipeline.nf -resume

# Generate execution report
$ nextflow run cholera_pipeline.nf -with-report report.html

Configuration File

nextflow.config

params {
reads = "data/*_{1,2}.fastq.gz"
outdir = "results"
}

process {
cpus = 4
memory = '8 GB'
}

docker {
enabled = true
}
