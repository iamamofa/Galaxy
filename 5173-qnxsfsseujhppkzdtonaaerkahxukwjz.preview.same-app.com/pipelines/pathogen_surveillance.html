Introduction

nf-core/pathogensurveillance is a Nextflow pipeline for pathogen identification, variant detection, and biosurveillance. The pipeline generates interactive HTML reports, supports multiple compute infrastructures, and uses Docker, Singularity, or Conda for reproducible installation.

Key Features

  • Population genomics analysis for multiple organisms
  • Rapid k-mer-based identification followed by phylogenetic analysis
  • Automated reference retrieval from NCBI RefSeq
  • Support for user-defined sample groupings and reports
  • Nextflow DSL2 with one container per process for maintainability
  • Continuous integration testing on AWS with full-sized datasets

Quick Start

Test your Nextflow and nf-core setup first using a small dataset:

nextflow run nf-core/pathogensurveillance -profile test,docker --outdir test_output

Test Dataset Profiles

Available test profiles for different organisms and dataset sizes:

  • test: 1 small genome for fast testing
  • test_serratia: 10 Serratia isolates
  • test_bordetella: 5 Bordetella pertussis isolates
  • test_salmonella: 5 Salmonella isolates
  • test_boxwood_blight: 5 samples of Cylindrocladium buxicola
  • test_mycobacteroides: 5 Mycobacteroides abscessus samples
  • test_bacteria: 10 mixed bacteria
  • test_klebsiella: 10 K. pneumoniae and related species
  • test_small_genomes: 6 samples with small genomes

To run a test profile with Docker:

nextflow run nf-core/pathogensurveillance -profile docker,test_bacteria -resume --outdir test_output

Sample sheets for these profiles can be found here:

nf-core test datasets

Running Your Own Data

nextflow run nf-core/pathogensurveillance -profile <docker|singularity|conda> -resume --input <samplesheet.tsv/csv> --outdir <output_path>
