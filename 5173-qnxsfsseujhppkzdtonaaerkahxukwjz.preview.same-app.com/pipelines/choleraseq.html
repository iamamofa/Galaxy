Introduction

CERI-KRISP/CholeraSeq is a Nextflow pipeline for genomic data analysis of Cholera outbreaks. It supports reproducible analysis using Docker, Singularity, or Conda environments.

Reference Dataset

A multi-fasta reference including global cohorts is available on NCBI:

Zenodo Dataset

Installation

Run CholeraSeq using Nextflow with Docker, Singularity, or Conda. Example using Docker for testing:

$ nextflow run CERI-KRISP/CholeraSeq -profile test,docker --outdir test_output

For Singularity:

$ nextflow run CERI-KRISP/CholeraSeq -profile test,singularity --outdir test_output

Samplesheet Input

Create a CSV samplesheet with three columns (header required):

sample,fastq_1,fastq_2
SRR8364252,ftp://ftp.sra.ebi.ac.uk/..._1.fastq.gz,ftp://ftp.sra.ebi.ac.uk/..._2.fastq.gz
SRR771360,ftp://ftp.sra.ebi.ac.uk/...fastq.gz,
AHGB01000000.1,https://github.com/.../AHGB01.fasta,

Specify the samplesheet when running the pipeline:

--samplesheet '[path to samplesheet file]'

Running the Pipeline

Example command using Docker profile:

nextflow run CERI-KRISP/CholeraSeq \
         -profile docker \
         --samplesheet /path/to/samplesheet.csv \
         --outdir results

Optional: Use a YAML/JSON file to repeatedly specify parameters:

nextflow run ceri-krisp/choleraseq -profile docker -params-file params.yaml

Updating & Reproducibility

  • Pull latest version: nextflow pull ceri-krisp/choleraseq
  • Specify pipeline version for reproducibility: -r 1.3.1
  • Cached results are reused with -resume flag

Core Nextflow Arguments

  • -profile: Choose configuration profile (docker, singularity, conda, etc.)
  • -resume: Resume previous runs using cached results
  • -c: Specify custom config file (for resource tuning)

Resources & Customization

CholeraSeq allows custom resource requests, containers, and tool arguments. For more details, consult the nf-core documentation and nf-core/configs repository.
