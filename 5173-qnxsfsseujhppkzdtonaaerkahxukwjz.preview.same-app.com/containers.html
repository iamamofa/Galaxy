Containers in Bioinformatics

Docker and Singularity

Containers package software with all its dependencies, ensuring reproducibility across different computing environments. They're essential for modern bioinformatics workflows.

What are Containers?

  • Lightweight, standalone packages containing everything needed to run software
  • Include code, runtime, system tools, libraries, and settings
  • Run identically on any system that supports containers
  • More portable than virtual machines, less overhead

Docker Basics

Installing Docker

# On Ubuntu/Debian
$ curl -fsSL https://get.docker.com -o get-docker.sh
$ sudo sh get-docker.sh

# Add user to docker group
$ sudo usermod -aG docker $USER

# Test installation
$ docker run hello-world

Running Bioinformatics Tools in Docker

# Pull FastQC container from BioContainers
$ docker pull biocontainers/fastqc:v0.11.9_cv8

# Run FastQC on your data
$ docker run -v $(pwd):/data biocontainers/fastqc:v0.11.9_cv8 \
fastqc /data/sample.fastq

-v flag: Mounts your current directory to /data inside container

SPAdes Genome Assembly Example

# Pull SPAdes container
$ docker pull quay.io/biocontainers/spades:3.15.5--h95f258a_1

# Run assembly
$ docker run -v $(pwd):/data quay.io/biocontainers/spades:3.15.5--h95f258a_1 \
spades.py -1 /data/R1.fastq -2 /data/R2.fastq -o /data/assembly

Creating Custom Docker Images

Example Dockerfile

Create a Dockerfile for your custom analysis pipeline:

FROM ubuntu:20.04

# Install dependencies
RUN apt-get update && apt-get install -y \
python3 python3-pip wget \
&& rm -rf /var/lib/apt/lists/*

# Install bioinformatics tools
RUN pip3 install biopython numpy pandas

# Set working directory
WORKDIR /analysis

# Copy analysis scripts
COPY scripts/ /analysis/scripts/

CMD ["/bin/bash"]

Build and Run Custom Image

# Build image
$ docker build -t cholera-analysis:v1.0 .

# Run container
$ docker run -it -v $(pwd):/data cholera-analysis:v1.0

Singularity for HPC

Singularity is preferred on HPC systems as it doesn't require root privileges and integrates better with scheduling systems.

Converting Docker to Singularity

# Build Singularity image from Docker
$ singularity build fastqc.sif docker://biocontainers/fastqc:v0.11.9_cv8

# Run with Singularity
$ singularity exec fastqc.sif fastqc sample.fastq

Bind Mounting in Singularity

$ singularity exec --bind /scratch:/data fastqc.sif \
fastqc /data/sample.fastq

Practical Exercise

Exercise: Run Cholera Analysis in Container

# 1. Pull FastQC container
$ docker pull biocontainers/fastqc:v0.11.9_cv8

# 2. Create test directory
$ mkdir -p ~/cholera_test
$ cd ~/cholera_test

# 3. Run quality control
$ docker run -v $(pwd):/data biocontainers/fastqc:v0.11.9_cv8 \
fastqc /data/*.fastq

Best Practices

  • Use official biocontainers from trusted repositories
  • Pin specific versions for reproducibility
  • Document your container versions in your methods
  • Use .dockerignore to exclude unnecessary files
  • Keep containers small - use minimal base images
